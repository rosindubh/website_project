Git - A distributed version-control system for tracking changes

Installing Git

On Linux enter [apt-get install git] at the terminal

Configuring Git

The first thing you should do after you install Git is to set your user name and e-mail address.

This is important because every Git commit uses this information. and it’s immutably is baked into the commits you start creating.


You do this by entering the following commands into a terminal:

$ git config --global user.name "John Doe"

$ git config --global user.email johndoe@example.com

In the above examples you would replace [johndoe] and [johndoe@xample.com] with your own username and email address.


Now that your identity is set up, you can configure the default text editor that will be used.

If not configured, Git uses your system’s default editor which is generally Vim.

If you want to use a different text editor such as Emacs, you can do the following:

$ git config --global core.editor emacs


If you want to check your settings, you can use the git config --list command to list all the settings Git can find at that point:

$ git config --list

You can also check what Git thinks a specific key’s value is by typing:

$ git config user.name


If you ever need help while using Git, there are three ways to get the manual

$ git help [verb]

$ git [verb] --help

$ man git [verb]

For example, you can get the manpage help for the config command by running

$ git help config
